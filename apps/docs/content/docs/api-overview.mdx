The OpenAPI document is the source of truth for the exact request/response schema of every endpoint,
public and admin:

- [`apps/gateway/openapi.yaml`](https://github.com/boelabs/unified-gateway/blob/main/apps/gateway/openapi.yaml) — OpenAPI 3.1. Importable into Bruno, Postman, Insomnia, and similar tools.

This page covers what's shared across every `/v1/*` endpoint; each endpoint's own page covers its
specific contract.

## Authentication

- **Inference** (`/v1/*`): master key or virtual key, via `Authorization: Bearer <key>`. Alternatives:
  the `x-api-key` header, or `?api_key=<key>` in the query string (for `EventSource`, which can't set
  custom headers). `/v1/models` and `/v1/models/{model}` are the only unauthenticated exceptions — see
  [Model discovery](/docs/models-discovery).
- **Admin** (`/admin/*`): requires the master key.

Full model: [Security](/docs/security).

## Error shape

Every error uses the OpenAI-compatible envelope, translated to Anthropic's shape on `/v1/messages`:

```json
{ "error": { "message": "...", "type": "invalid_request_error", "param": "model", "code": "model_not_found" } }
```

The client-facing `message` is always a stable, generic-per-class string — the gateway is a router, so
the same Public Model can fail differently on different attempts, and the public error can't leak a
specific provider's wording or leak internals. `type`, `code`, and `param` give actionable detail
without doing that. See [Troubleshooting](/docs/troubleshooting) for the full class/status/code table.

## Request correlation

Every response carries `x-request-id` (echoed if you send one). **Always include it** when reporting an
issue — see [Headers](/docs/headers) for the full header reference and
[Observability](/docs/observability) for how to look one up in the logs.

## Importing into Bruno

**Import Collection → OpenAPI V3**, select `apps/gateway/openapi.yaml`, then set the `baseUrl` server
variable (e.g. `http://localhost:4000`) and the bearer token in Bruno's auth UI.

## Typical flow to serve a model

1. `POST /admin/deployments` — create a deployment: `publicModel` + `adapterKey` + `upstreamModel` + inline credentials.
2. `POST /admin/keys` — issue a virtual key with `allowedModels`.
3. Call an inference endpoint with the public model name in `model`.

`POST /admin/deployments/resolve` validates the profile/operations/transports without saving — see
[First deployment](/docs/first-deployment).
