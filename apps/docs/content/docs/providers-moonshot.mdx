---
title: "Moonshot"
description: "Adapter key moonshot. Built-in base URL, no provider preset needed."
---

```json
{ "publicModel": "kimi", "adapterKey": "moonshot", "upstreamModel": "kimi-k2.7-code", "credentials": { "apiKey": "..." } }
```

- **Credentials**: `apiKey` only. No `baseUrl` needed — defaults to `https://api.moonshot.ai/v1`.
- Use `adapterKey: "moonshot"` (there's no `provider` preset for this one).
- **Default transport**: Chat Completions.
- **Reasoning**: native Kimi thinking, `openai_body` kind for most models; some coding-focused variants
  (e.g. `kimi-k2.7-code`) are declared `fixed` — always-on reasoning with no off switch, since that
  variant has no documented way to disable it.
- Catalog: `src/adapters/moonshot/catalog.json`.
